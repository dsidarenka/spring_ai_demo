GET /tools/chat

Tool Calling

The LLM can autonomously invoke registered Java methods to fetch data it does not know. The model decides when to call a tool and what arguments to pass — your code never drives the loop manually.

Registered tools
getCurrentWeather(city)
Get the current weather conditions and temperature for a specified city
getWeatherForecast(city, days)
Get a multi-day weather forecast (1–5 days) for a specified city

GET /recursive/chat

Recursive Advisor (ToolCallAdvisor)

ToolCallAdvisor is Spring AI's built-in recursive advisor. Unlike simple tool calling — where the ChatClient delegates execution to the model and stops — this advisor owns the loop: it disables the model's internal tool execution and drives the full Thought → Action → Observation cycle itself, repeating until the LLM produces a final answer with no outstanding tool calls.

↓
LLM (Reasoning)
←

↻ loop while tool calls present
↓ no more tool calls
Final Answer

The same weather tools from the Tool Calling tab are registered. Try prompts that naturally require multiple tool calls — the advisor will keep looping until all facts are gathered.
